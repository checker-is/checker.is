🙾 Checker - coin stabilization technology for Tezos.

FAQ

  • What is Checker?

    Checker is a software project implementing financial logic designed to help stabilize the value of a cryptographic coin with respect to an externally provided index.

  • Is Checker a token or a coin of some sort? Is it a stablecoin?

    No, Checker is a software project. The code base does define three different kinds of tokens and implements financial logic designed to stabilize the value of one of them with respect to an externally provided index or reference.

  • Is Checker a protocol amendment for Tezos?

    No, Checker's core logic is articulated around a set of smart-contracts.

  • Is there a white paper or source code available?

    No, the details are still stealth.

  • Who is developing Checker?

    Currently, Arthur Breitman is developing Checker.

  • Does Checker implement an algorithmic stablecoin?

    There doesn't seem to be an agreed upon definition on what that means. The closest existing project in terms of design is MakerDAO.

  • Is Checker like MakerDAO?

    There are many similarities, but also important design differences. For instance, there is no reliance on a global settlement mechanism, and the terms under which collateral is pledged are more flexible.

  • If such a software were deployed, couldn't this create a concentration of baking power?

    No. The system is designed such that, when tez is pledged, the baking rights are retained by the party pledging the collateral.

  • Is Checker like Tether

    No.

  • Is Checker like Basis?

    No.

  • Broadly speaking, how does it work?

    Tokens are stabilized by backing them, on chain, with contracts collateralized by a cryptocurrency and indexed on an external metric provided by a set of oracles.

  • Are you interested in designing a multi-collateral based system?

    Not particularly. If the collateral is a claim on some off-chain assets, then you lose the censorship resistance aspect of an on-chain system. Purely on-chain assets that have any real value and aren't a base layer cryptocurrency are pretty rare.

  • How can you get parity without convertibility?

    You can't. As a result, a possible failure mode of the system is that the total supply of stabilized coins goes to 0. This is not a particularly dramatic outcome.

  • Does the financial logic guarantee the stability of a stablecoin?

    No, that's impossible. In such systems there's always some residual risk. That said, this isn't merely trading off volatility for tail risk.

  • If there is a risk, there should be a risk discount, so how can there be a peg?

    There is also a convenience premium. For some amount of supply, they cancel each other out.

  • Why is it called Checker

    Checkers are simpler than chess, and the system is meant to be simple to understand and analyze. In addition, the name is easy to remember and provides for interesting visual components around checkerboard patterns.
